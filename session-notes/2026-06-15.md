### GRACE animation: 12×14 pixel output
First export came back at 12×14 pixels. Root cause: GEE ignored `dimensions:800` and exported at native GRACE resolution (~55km/pixel over the SW Colorado bbox = ~12×14 native pixels).

**Fix 1** — Added `resample('bicubic').reproject({crs:'EPSG:4326', scale:10000})` before `.visualize()` in each frame. Forces GEE to work at 10km resolution so `dimensions:800` produces a proper-sized video. Bicubic smooths the coarse GRACE pixels rather than leaving blocky squares.

**Fix 2** — Expanded region from `swColorado` to `fourCorners` (`[-115.0, 31.0, -101.5, 42.5]`) covering AZ, CO, NM, UT with margins. GRACE is too coarse (~5-6 pixels over SW Colorado) to show meaningful spatial variation at that scale; Four Corners gives ~30+ pixels and makes drought patterns legible.

**Fix 3** — Added TIGER/2018/States boundary lines baked into each frame:
```javascript
var stateBoundaries = ee.FeatureCollection('TIGER/2018/States');
var stateMask = ee.Image().byte()
  .paint({featureCollection: stateBoundaries, color: 1, width: 1});
// In graceFrames map:
return graceViz.where(stateMask, ee.Image.constant([255, 255, 255]).byte());
```
White 1px lines. Can bump to width:2 if hard to see against mid-palette neutrals.

Updated `grace.filterBounds` and `graceAnomalies.clip` to use `fourCorners`. GRACE CSV time series still reduces over `swColorado` geometry.

Export.video region changed to `fourCorners`; CSV exports unchanged.
